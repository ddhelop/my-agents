# Studio AI 에이전트 컬렉션

빠른 개발의 모든 측면을 가속화하고 향상시키도록 설계된 전문 AI 에이전트들의 종합 컬렉션입니다. 각 에이전트는 해당 도메인의 전문가로서, 전문 지식이 필요할 때마다 호출될 준비가 되어 있습니다.

## 📥 설치 방법

1. **이 저장소 다운로드:**

   ```bash
   git clone https://github.com/your-repo/my-agents.git
   ```

2. **Claude Code 에이전트 디렉토리에 복사:**

   ```bash
   cp -r my-agents/* ~/.claude/agents/
   ```

   또는 모든 에이전트 파일을 `~/agents/` 디렉토리에 수동으로 복사하세요.

3. **Claude Code를 재시작**하여 새로운 에이전트들을 로드하세요.

## 🚀 빠른 시작

에이전트들은 Claude Code에서 자동으로 사용 가능합니다. 작업을 설명하면 적절한 에이전트가 트리거되거나, 에이전트 이름을 직접 언급할 수 있습니다.

📚 **더 알아보기:** [Claude Code 서브 에이전트 문서](https://docs.anthropic.com/en/docs/claude-code/sub-agents)

### 사용 예시

- "명상 습관 추적 앱을 새로 만들어줘" → `프론트엔드-개발자`
- "우리 앱 리뷰가 떨어지고 있어, 뭐가 문제야?" → `QA-테스터`
- "이 로딩 화면을 더 재미있게 만들어줘" → `인터랙션-디자이너`
- "AI 기능을 추가하고 싶어" → `AI-개발자`

## 📋 전체 에이전트 목록

### 엔지니어링 부서 (`engineers/`)

- **프론트엔드-개발자** - 실제로 배포되는 초고속 사용자 인터페이스 구축
- **백엔드-개발자** - 확장 가능한 API와 서버 시스템 설계
- **데브옵스-개발자** - 문제없이 지속적으로 배포
- **AI-개발자** - 실제로 배포되는 AI/ML 기능 통합
- **크로스플랫폼-개발자** - 네이티브 iOS/Android 경험 제작
- **테스터** - 실제 버그를 잡는 테스트 작성

### 제품 부서 (`product-managers/`)

- **프로덕트-매니저** - 6일 안에 최대 가치 배송

### 디자인 부서 (`designers/`)

- **UI/UX-디자이너** - 개발자가 실제로 구축할 수 있는 인터페이스 설계
- **인터랙션-디자이너** - 모든 상호작용에 즐거움 추가

### 테스팅 부서 (`testers/`)

- **QA-테스터** - API가 압박 상황에서도 작동하도록 보장

## 🎯 자동 실행 에이전트

특정 상황에서 자동으로 트리거되는 에이전트들:

- **테스터** - 기능 구현, 버그 수정, 또는 코드 수정 후
- **인터랙션-디자이너** - UI/UX 변경 후
- **QA-테스터** - 기능 구현 완료 후 품질 검증

## 💡 모범 사례

1. **에이전트들이 함께 작업하도록 하세요** - 많은 작업이 여러 에이전트로부터 혜택을 받습니다
2. **구체적으로 설명하세요** - 명확한 작업 설명이 에이전트 성능을 향상시킵니다
3. **전문성을 신뢰하세요** - 에이전트들은 특정 도메인을 위해 설계되었습니다
4. **빠르게 반복하세요** - 에이전트들은 6일 스프린트 철학을 지원합니다

## 🔧 기술 세부사항

### 에이전트 구조

각 에이전트는 다음을 포함합니다:

- **name**: 고유 식별자
- **description**: 예시와 함께 언제 사용해야 하는지
- **color**: 시각적 식별
- **tools**: 에이전트가 접근할 수 있는 특정 도구들
- **시스템 프롬프트**: 상세한 전문성과 지침

### 새 에이전트 추가하기

1. 적절한 부서 폴더에 새 `.md` 파일 생성
2. YAML frontmatter로 기존 형식 따르기
3. 3-4개의 상세한 사용 예시 포함
4. 포괄적인 시스템 프롬프트 작성 (500단어 이상)
5. 실제 작업으로 에이전트 테스트

## 🛠️ 스튜디오에 맞게 에이전트 커스터마이징

### 에이전트 커스터마이징 할 일 목록

특정 요구사항에 맞게 에이전트를 생성하거나 수정할 때 이 체크리스트를 사용하세요:

#### 📋 필수 구성요소

- [ ] **YAML Frontmatter**
  - [ ] `name`: 고유 에이전트 식별자 (kebab-case)
  - [ ] `description`: 언제 사용하는지 + 컨텍스트/해설이 있는 3-4개의 상세한 예시
  - [ ] `color`: 시각적 식별 (예: blue, green, purple, indigo)
  - [ ] `tools`: 에이전트가 접근할 수 있는 특정 도구들 (Write, Read, MultiEdit, Bash 등)

#### 📝 시스템 프롬프트 요구사항 (500단어 이상)

- [ ] **에이전트 정체성**: 명확한 역할 정의와 전문 영역
- [ ] **핵심 책임**: 5-8개의 구체적인 주요 업무
- [ ] **도메인 전문성**: 기술 스킬과 지식 영역
- [ ] **스튜디오 통합**: 에이전트가 6일 스프린트 워크플로우에 맞는 방법
- [ ] **모범 사례**: 구체적인 방법론과 접근법
- [ ] **제약사항**: 에이전트가 해야 할/하지 말아야 할 것
- [ ] **성공 지표**: 에이전트 효과성을 측정하는 방법

#### 🎯 에이전트 유형별 필수 예시

**엔지니어링 에이전트**는 다음에 대한 예시가 필요합니다:

- [ ] 기능 구현 요청
- [ ] 버그 수정 시나리오
- [ ] 코드 리팩토링 작업
- [ ] 아키텍처 결정

**디자인 에이전트**는 다음에 대한 예시가 필요합니다:

- [ ] 새 UI 컴포넌트 생성
- [ ] 디자인 시스템 작업
- [ ] 사용자 경험 문제
- [ ] 시각적 정체성 작업

**제품 에이전트**는 다음에 대한 예시가 필요합니다:

- [ ] 기능 우선순위 결정
- [ ] 사용자 피드백 분석
- [ ] 시장 조사 요청
- [ ] 전략 계획 필요

#### ✅ 테스팅 & 검증 체크리스트

- [ ] **트리거 테스팅**: 의도된 사용 사례에 대해 에이전트가 올바르게 활성화됨
- [ ] **도구 접근**: 에이전트가 지정된 모든 도구를 올바르게 사용할 수 있음
- [ ] **출력 품질**: 응답이 도움이 되고 실행 가능함
- [ ] **엣지 케이스**: 예상치 못한 또는 복잡한 시나리오를 처리함
- [ ] **통합**: 다중 에이전트 워크플로우에서 다른 에이전트들과 잘 작동함
- [ ] **성능**: 합리적인 시간 내에 작업을 완료함
- [ ] **문서화**: 예시가 실제 사용 패턴을 정확히 반영함

#### 🔧 에이전트 파일 구조 템플릿

```markdown
---
name: your-agent-name
description: 이 에이전트는 [시나리오]일 때 사용하세요. 이 에이전트는 [전문성]을 전문으로 합니다. 예시:\n\n<example>\n상황: [상황]\n사용자: "[사용자 요청]"\n어시스턴트: "[응답 접근법]"\n<commentary>\n[이 예시가 중요한 이유]\n</commentary>\n</example>\n\n[3개 더 예시...]
color: agent-color
tools: Tool1, Tool2, Tool3
---

당신은 [역할]을 하는 [주요 기능]입니다. 당신의 전문성은 [도메인]에 걸쳐 있습니다. 6일 스프린트에서 [스프린트 제약]임을 이해하므로, 당신은 [접근법]합니다.

주요 책임:

1. [책임 1]
2. [책임 2]
   ...

[상세한 시스템 프롬프트 내용...]

당신의 목표는 [궁극적인 목적]입니다. 당신은 [핵심 행동 특성]합니다. 기억하세요: [6일 스프린트를 위한 핵심 철학].
```

#### 📂 부서별 가이드라인

**엔지니어링** (`engineers/`): 구현 속도, 코드 품질, 테스팅에 집중
**디자인** (`designers/`): 사용자 경험, 시각적 일관성, 빠른 반복 강조
**제품** (`product-managers/`): 사용자 가치, 데이터 기반 결정, 시장 적합성 우선시
**테스팅** (`testers/`): 품질 보장, 병목 지점 찾기, 성능 검증
